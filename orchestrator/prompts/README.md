# Prompts ‚Äî Design Notes

This directory contains the frozen decision schema and system prompt used by all three LLM traders.
Phase 0 artifacts. **Do not change `schema.json` once Phase 1 begins** (IFACE-03).

---

## Files

| File | Purpose |
|------|---------|
| `schema.json` | Canonical Draft 2020-12 validation schema (validation variant, `additionalProperties:true`) |
| `system.md` | Identical system prompt for all three models (Jinja2 template; placeholders filled at runtime) |
| `README.md` | This file ‚Äî design notes on dual-schema derivation, provider shapes, and journal metadata |

## Dual-Schema Derivation (Pitfall 1)

Two schema variants are needed, but they MUST derive from ONE canonical source (`schema.json`):

### Variant 1 ‚Äî Validation (canonical, stored in `schema.json`)

- `additionalProperties: true`
- Required: `action`, `sizeUsd`, `leverage`, `rationale`, `confidence`, `expectedHoldingPeriod`
- `market` and `side` are optional (absent on `hold` / `close`)
- Purpose: forward-compat validation of model output (D-08). Extra fields accepted + stored verbatim in journal; ignored for trade execution.

### Variant 2 ‚Äî Provider-strict (generated at runtime by `schema.py`)

- `additionalProperties: false`
- **All** property keys listed in `required`
- Generated by `orchestrator.schema.strict_provider_schema()` ‚Äî a deep copy of the canonical schema with `additionalProperties` flipped and `required` expanded to all properties.
- Purpose: fed to provider structured-output APIs (D-09). Forces the model to emit only declared fields.

**Never hand-maintain two separate schema files.** Always call `strict_provider_schema()` to obtain the provider-strict variant.

## Three Provider Request Shapes (D-09 / Phase 2 wiring notes)

Phase 0 freezes the schema. Phase 2 wires these shapes into the orchestrator.

### Anthropic ‚Äî claude-opus-4-7

```python
# Tool-use forced-tool pattern with strict schema
# CRITICAL: DO NOT pass temperature ‚Äî returns HTTP 400 on Opus 4.7
response = client.messages.create(
    model="claude-opus-4-7",
    # temperature=...  <-- OMIT ENTIRELY
    tools=[{
        "name": "submit_decision",
        "description": "Submit your trading decision for this cycle.",
        "input_schema": strict_provider_schema(),  # additionalProperties:false, all required
    }],
    tool_choice={"type": "tool", "name": "submit_decision"},
    messages=[{"role": "user", "content": rendered_prompt}],
)
# Response in response.content[0].input (dict)
```

**Note on replay:** Claude Opus 4.7 uses adaptive sampling ‚Äî responses are NOT byte-reproducible.
The verifier reports `SEMANTIC_MATCH` (decision payload equivalent) vs `MISMATCH` (payload differs),
not byte equality. See `D-75` and verifier design in Phase 5.

### OpenAI ‚Äî gpt-5.5-2026-04-23

```python
# Structured output with json_schema strict mode
response = client.chat.completions.create(
    model="gpt-5.5-2026-04-23",
    temperature=0,
    seed=DETERMINISTIC_SEED,          # enables best-effort determinism
    reasoning_effort="none",           # omit reasoning overhead for fast trading decisions
    response_format={
        "type": "json_schema",
        "json_schema": {
            "name": "TraderDecision",
            "schema": strict_provider_schema(),   # additionalProperties:false, all required
            "strict": True,
        },
    },
    messages=[{"role": "system", "content": rendered_prompt}],
)
# Response in json.loads(response.choices[0].message.content)
```

### Google ‚Äî gemini-3.1-pro-preview

```python
# NEW SDK: google-genai (NOT google-generativeai ‚Äî deprecated)
from google import genai
from google.genai import types

client = genai.Client()
response = client.models.generate_content(
    model="gemini-3.1-pro-preview",
    contents=rendered_prompt,
    config=types.GenerateContentConfig(
        response_mime_type="application/json",
        response_schema=strict_provider_schema(),   # additionalProperties:false, all required
        temperature=0,
        seed=DETERMINISTIC_SEED,
    ),
)
# Response in json.loads(response.text)
```

## Server-Captured Journal Metadata (D-07)

The following fields are added by the orchestrator to each journal entry. They are NOT emitted
by the model and must NOT appear in `schema.json`:

| Field | Source | Description |
|-------|--------|-------------|
| `reasoningTokens` | Provider API usage block | Tokens used for reasoning/thinking (where exposed). Claude: `usage.cache_creation_input_tokens`; OpenAI: `usage.completion_tokens_details.reasoning_tokens`; Gemini: `usageMetadata.cachedContentTokenCount`. |
| `responseLatencyMs` | Server wall-clock | `time.monotonic()` delta from request send to response received, in milliseconds. |

These are observability and verifier metadata only. They have no influence on trade execution.

## Frozen Fields (D-06 ‚Äî Deferred)

`stopLoss` and `takeProfit` are intentionally absent from `schema.json`. They are deferred to
Phase 1 ‚Äî the vault may or may not enforce them at the perps interaction layer. Do not add these
fields to `schema.json` until the Phase 1 operator decision is made.
